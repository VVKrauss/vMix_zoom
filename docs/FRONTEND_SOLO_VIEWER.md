# Соло-страница: поведение UI (фронт)

Документ для разработчиков и поддержки. Контракт API для бэка — в [`BACKEND_SOLO_VIEWER.md`](./BACKEND_SOLO_VIEWER.md).

## Полноэкранный режим

- После успешного подключения показывается **только видео** на весь viewport (`position: fixed`, чёрный фон).
- **Звук** идёт в **том же** элементе `<video>`: поток собирается из consumer’ов (дорожки video + audio в одном `MediaStream`).
- Отдельной шапки, подписей с `peerId`/комнатой и отдельного `<audio>` нет.

## Управление

- **Escape** — выход с соло-страницы: из URL убирается `peer`, остаётся `?room=…`, открывается обычный экран входа (см. логику в `App.tsx`).
- **Клик** по области видео вызывает `video.play()` повторно — на случай, если браузер **заблокировал автозапуск со звуком** до пользовательского жеста.

## Состояния до/после потока

- **Подключение** — чёрный экран и спиннер по центру.
- **Нет ни видео, ни аудио** — текст «Ожидание потока…» по центру (без панелей).
- **Ошибка** / **участник вышел** — короткое сообщение и кнопки «Повторить» / «Назад» (иначе нельзя выйти без Escape/истории браузера).

## Несколько вкладок с одной соло-ссылкой

**В коде фронта нет ограничения «один экземпляр соло на браузер».** Каждая вкладка — свой `Socket.IO`, свой `joinRoomAsViewer`, свои transports/consumers.

Если **вторая вкладка не показывает картинку, пока открыта первая**, это **не заложенное поведение** этого репозитория. Имеет смысл проверить:

1. **Бэкенд** — нет ли лимита «один зритель на `watchPeerId`», общего mutex на комнату или ошибок во втором `joinRoomAsViewer`/`consume` (логи сервера для второго сокета).
2. **Фокус вкладки** — в фоне Chrome может **замедлять таймеры и сеть**; для проверки откройте вторую вкладку и сделайте её **активной**.
3. **Ресурсы** — на слабой машине два декодера H.264 могут упираться в лимиты (реже).

Итого: для типичного SFU **несколько независимых зрителей** (несколько вкладок или устройств) на один и тот же producer — нормальная схема; зависимость от закрытия первой вкладки чаще указывает на **сервер или окружение**, а не на описанный здесь фронт.
